# Security Policy

## Supported versions

The latest released version receives security fixes. Please upgrade to the
latest release before reporting an issue.

## Reporting a vulnerability

keel handles SSH access, sudoers, and credentials, so we take reports seriously.

**Please do not open a public issue for a security vulnerability.**

Use GitHub's private vulnerability reporting:
**Security → Report a vulnerability** on
<https://github.com/DigitalTolk/keel/security/advisories/new>.

Include the version, your platform, reproduction steps, and the impact. We'll
acknowledge your report, keep you updated on the fix, and credit you in the
release notes unless you prefer otherwise.
